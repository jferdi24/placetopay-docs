# API de Administración y Gestión de Tokens

Las APIs de **Token Requestor** proporcionan una infraestructura segura y flexible para la tokenización de tarjetas de pago y la gestión de los recursos asociados.
Su objetivo principal es garantizar un **procesamiento de pagos seguro** y ofrecer a los administradores un **control centralizado** sobre consumidores, comercios, sitios y credenciales.

Estas APIs están diseñadas para integrarse con pasarelas de pago, proveedores de servicios y comercios, brindando herramientas para:
- Reducir riesgos asociados al manejo de datos sensibles.
- Optimizar la experiencia de usuario en procesos de tokenización.
- Mantener un ecosistema escalable y alineado con los estándares de las marcas de pago (Visa, Mastercard).

# Arquitectura y Diseño del Sistema

## Componentes Principales

- **API de Tokenización**
  Ofrece diferentes endpoints responsables de:
  - Generación de tokens
  - Consulta
  - Actualización
  - Eliminación
  - Creación de criptogramas a partir de tokens activos

  Todo esto permite el **procesamiento seguro de pagos**.

- **API de Administración**
  Permite la **gestión y supervisión** de:
  - Consumidores
  - Comercios
  - Sitios
  - Otros recursos asociados al sistema de tokenización

  A través de esta API, los administradores pueden:
  - Gestionar consumidores, comercios y sitios
  - Administrar credenciales y accesos a las demás APIs
