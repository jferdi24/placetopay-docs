# Códigos de razón

El siguiente apartado describe los posibles códigos de respuesta que puede retornar el sistema en las distintas operaciones de tokenización, validación y gestión de tarjetas. Estos códigos ayudan a comprender el estado de una operación y permiten identificar con claridad posibles errores o condiciones especiales.

| Código | Descripción                                                                                                                               |
|--------|-------------------------------------------------------------------------------------------------------------------------------------------|
| 00     | Operación exitosa. Indica que el flujo fue procesado correctamente sin errores.                                                           |
| BR     | La solicitud contiene errores, puede tener datos inválidos, incumplir reglas de validación o referenciar recursos no autorizados.         |
| 15     | La marca de la tarjeta no está habilitada para el comercio.                                                                               |
| 31     | El banco emisor no está soportado por el servicio.                                                                                        |
| 56     | El identificador del cliente o comercio no existe en el sistema de la marca.                                                              |
| 62     | La tarjeta no cumple con las reglas definidas por la red para ser tokenizada.                                                             |
| 14     | El número de tarjeta no es válido o no es elegible para tokenización, por fallas de formato o políticas de la marca.                      |
| 55     | Fallo de autenticación, las credenciales o llaves son inválidas o están mal configuradas.                                                 |
| ?I     | No se encontró el token o la tarjeta, la información consultada no existe en el sistema de la marca.                                      |
| XF     | El comercio no tiene habilitada la franquicia correspondiente para operar.                                                                |
| SU     | El servicio no está habilitado, puede deberse a que la marca no está activa o el proceso de onboarding no ha sido culminado.              |
| X3     | Error en la solicitud enviada a servicios externos, puede tratarse de formato incorrecto, campos inválidos o datos incompletos.           |
| XN     | Error con la tarjeta, puede estar deshabilitada ante el emisor o no cumplir validaciones generales requeridas.                            |
| XR     | La respuesta recibida desde la marca o servicio externo no fue reconocida o tuvo un formato inesperado.                                   |
| RM     | Se superó el límite de solicitudes permitido, error 429 por demasiadas peticiones en un corto periodo de tiempo.                          |
| XH     | Error interno inesperado en el sistema que impidió procesar la operación.                                                                 |
| ?-     | La operación se encuentra en estado pendiente, aplica a flujos asincrónicos que aún no han finalizado.                                    |
| 05     | Ha ocurrido un error durante el procesamiento con el servicio externo.                                                                    |
